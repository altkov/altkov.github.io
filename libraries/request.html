Request Library

Request library contains your request data.

Basically, it's just copies all the request parameters (like $_POST, $_GET) to the properties.

The only and main benefit from it is that library escapes all the values with clean($data) method, so it's safe to inject this values in html (like e-mail messages, or search params).
But not the database! Use escape method of your database engine before put something in database.
Also you should use double quotes and not the single quotes in your html attributes if you put there data from request library. That's because used htmlspecialchars escape method ENT_COMPAT not escapes single quotes.

That's why you should always use request properties and not superglobals to access request data.


You can call this library in controller with $this->request

Properties

public array get = array()
Contains cleaned $_GET data
public array post = array()
Contains cleaned $_POST data
public array cookie = array()
Contains cleaned $_COOKIE data
public array files = array()
Contains cleaned $_FILES data
public array server = array()
Contains cleaned $_SERVER data

Methods

public function clean(mixed $data) : mixed
Returns escaped with htmlspecialchars($data, ENT_COMPAT, 'UTF-8') data
If data is array, recursively escapes all the contained values (in child arrays too).
